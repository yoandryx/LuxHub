    **1. Create `vercel.json` at project root** with cron configuration per OPS-01:

    ```json
    {
      "crons": [
        {
          "path": "/api/escrow/enforce-timeouts",
          "schedule": "0 */6 * * *"
        }
      ]
    }
    ```

    Note: `0 */6 * * *` = every 6 hours (00:00, 06:00, 12:00, 18:00 UTC). Requires Vercel Pro plan. If on Hobby plan, this will silently degrade to once-daily. Since escrow timeouts are measured in days (7-30 day thresholds), once-daily is functionally acceptable.

    **2. Update `src/pages/api/escrow/enforce-timeouts.ts`** to support Vercel cron auth:

    The current code only checks `req.body.cronSecret`. Vercel cron sends `Authorization: Bearer <CRON_SECRET>` header AND uses GET method by default. Changes needed:

    a) Add GET method support alongside existing POST. At the top of the handler, change:
    ```typescript
    if (req.method !== 'POST') {
      return res.status(405).json({ error: 'Method not allowed' });
    }
    ```
    To:
    ```typescript
    if (req.method !== 'POST' && req.method !== 'GET') {
      return res.status(405).json({ error: 'Method not allowed' });
    }
    ```

    b) Update the auth check to also check the Authorization header. Replace the existing auth block:
    ```typescript
    const { adminWallet, cronSecret } = req.body;
    const isValidCron = cronSecret && cronSecret === process.env.CRON_SECRET;
    ```
    With:
    ```typescript
    const { adminWallet, cronSecret } = req.body || {};
    const authHeader = req.headers['authorization'] || req.headers['Authorization'];
    const isValidCron =
      (cronSecret && cronSecret === process.env.CRON_SECRET) ||
      (typeof authHeader === 'string' && authHeader === `Bearer ${process.env.CRON_SECRET}`);
    ```

    The `req.body || {}` handles GET requests where body is undefined.

    **3. OPS-05 (priority fee):** No code change needed. The existing `src/lib/solana/priorityFees.ts` already reads `NEXT_PUBLIC_PRIORITY_FEE_MICRO_LAMPORTS` and defaults to 50000. The env var will be set in Plan 04 during Vercel env cutover. Document in SUMMARY that OPS-05 is satisfied by existing code + env var.
