Implementation details:
- Two overlaid `<input type="range">` elements for dual-handle behavior
- Left input controls min value (clamped so it cannot exceed value[1])
- Right input controls max value (clamped so it cannot go below value[0])
- Track fill between handles via CSS `linear-gradient` on the track element, using percentage positions calculated from `(value - min) / (max - min) * 100`
- Two numeric `<input type="number">` fields below the slider for direct min/max entry
- `onChange` fires during drag (for visual preview), `onChangeCommitted` fires on `onPointerUp`/`onMouseUp`/`onTouchEnd` (for actual filtering)
- Numeric inputs fire `onChangeCommitted` on blur with 300ms debounce via setTimeout
- Number inputs show "SOL" suffix label text to the right of each input
- Wait -- the marketplace filters by USD (listingPriceUSD). Show "$" prefix, not "SOL". Min input placeholder: "Min", Max input placeholder: "Max"
