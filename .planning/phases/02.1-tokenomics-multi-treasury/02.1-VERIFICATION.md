### Gaps Summary

Two partial truths block full goal achievement:

**Gap 1 — Legal Language (TM-05, Warning):** The pool detail page (`pool/[id].tsx`) was comprehensively cleaned — STATUS_CONFIG labels use "Contributions", buttons say "Contribute", no prohibited words in JSX string literals. However, the `BagsPoolTrading` component rendered on the same page still has "Trade Pool Shares" (line 341) as a user-visible heading. This is a one-word fix (`Shares` -> `Tokens`) in a file not covered by Plan 04's scope.

**Gap 2 — Post-Graduation Wiring (TM-03, Minor):** The Bags webhook correctly records `bondingCurveActive: false` when TOKEN_GRADUATED fires. However, `pool/[id].tsx` does not pass `bondingCurveActive` as a prop to `BagsPoolTrading`, and `BagsPoolTrading` does not declare it in its Pool interface. The component gates trading on `tokenStatus === 'unlocked'` which works correctly for the trading flow, but the component cannot display distinct UI for the pre-graduation bonding curve phase vs post-graduation DEX phase. The core trading functionality is wired — the gap is that the component cannot reflect graduation state visually.

Both gaps are small in scope (single file, 1-5 line changes each) and do not block core tokenomics functionality. The full multi-treasury architecture, distribution flow, and pool UX sections are fully implemented and tested.
