    Props: `{ assetId?: string, vendorWallet: string, onComplete?: (poolId: string) => void, onCancel?: () => void, adminMode?: boolean }`

    **Admin mode (D-02):** When `adminMode={true}`:
    - Step 1 (Configure) shows an asset selector dropdown that fetches ALL available assets from `/api/assets?pooled=false` (not filtered by vendor). This allows admin to pick any vendor's listing to tokenize on their behalf.
    - The `assetId` prop is optional in admin mode -- admin selects it from the dropdown.
    - The vendorWallet is auto-populated from the selected asset's vendor record.
    - All other steps (fee-share, launch, confirm) work identically.

    **Standard mode (D-01):** When `adminMode` is falsy:
    - `assetId` is required (passed from vendor's listing).
    - Asset selector is hidden; shows selected asset info directly.
    - Vendor's own listings filtered by `pooled !== true`.

    4-step stepper per UI-SPEC: Configure > Sign Fee-Share > Launch Token > Confirm

    Step 1 -- Configure:
    - Display asset info (fetch from `/api/assets/${assetId}`)
    - Input fields: Token Name (default: asset brand + model), Token Symbol (default: first 4 chars of brand), Target Amount USD, Total Shares, Min Buy-In USD
    - Confirmation dialog per copywriting contract: "This will launch a token for this watch and remove the direct listing. Continue?"
    - On confirm: POST to `/api/pool/create` with asset info, then POST to `/api/bags/create-pool-token` with `{ poolId, adminWallet: vendorWallet, tokenName, tokenSymbol, step: 'setup' }`
    - Store returned `tokenMint` and `feeShareTxs` in component state

    Step 2 -- Sign Fee-Share:
    - Display list of fee-share transactions to sign (show count: "Sign {N} transactions")
    - For each tx: deserialize from base64, sign with wallet, send via connection.sendRawTransaction, wait for confirmation
    - Show progress: "Signing {i}/{N}..." with spinner per transaction
    - Use `FiLoader` (animated spin) while in progress, `FiCheckCircle` when confirmed, `FiAlertTriangle` if failed
    - On all confirmed: auto-advance to Step 3

    Step 3 -- Launch Token:
    - Call `/api/bags/create-pool-token` with `{ poolId, step: 'launch' }`
    - Sign returned `launchTx`, send, wait for confirmation
    - Show "Launching bonding curve..." with spinner
    - On confirmed: auto-advance to Step 4

    Step 4 -- Confirm:
    - Show success state: "Pool Created Successfully!"
    - Display: token mint address (truncated with copy button), pool link to `/pools/${poolId}`
    - "View Pool" button linking to the new pool detail page

    Error handling: if any step fails, show error message with retry button. Do NOT auto-advance on failure.
