    **1. Create `/api/pool/claim-distribution.ts`** (per D-10 claimable model):

    POST endpoint accepting `{ poolId, claimerWallet }`.

    Flow:
    1. Validate poolId and claimerWallet are present
    2. `dbConnect()`, find pool by ID
    3. Verify pool status is `distributing` or `sold` (with distributionStatus = 'proposed' or 'approved')
    4. Find claimer in `pool.distributions[]` by matching `wallet` field (case-insensitive comparison)
    5. Verify claimer has NOT already claimed (check `txSignature` is falsy and `distributedAt` is falsy)
    6. Verify claim window has not expired: if `pool.claimWindowExpiresAt` exists and `new Date() > pool.claimWindowExpiresAt`, return 400 "Claim window has expired"
    7. Return the claimable amount and distribution details: `{ claimableAmount: entry.amount, ownershipPercent: entry.ownershipPercent, wallet: entry.wallet, poolId }`
    8. If request includes `execute: true`, mark the distribution entry as claimed: set `txSignature = 'pending'` and `distributedAt = new Date()`. The actual SOL transfer is handled client-side (wallet signs the transfer). After client confirms TX, a follow-up PATCH call updates `txSignature` with the real signature.

    Add a PATCH handler on the same endpoint for confirming claim TX:
    - Body: `{ poolId, claimerWallet, txSignature }`
    - Updates the distribution entry's `txSignature` field
    - If ALL distributions have `txSignature` set OR `claimWindowExpiresAt` has passed, call a `finalizePool()` helper that:
      1. Executes on-chain SPL token burn via `@solana/spl-token` `burn()` + `closeAccount()` for the pool's token mint authority (per D-11: tokens are BURNED on-chain, not just flagged in MongoDB). Use the admin/treasury keypair loaded from env (`DEPLOY_KEYPAIR` or equivalent) to sign the burn TX. If the pool treasury holds remaining tokens, burn those first.
      2. Sets `pool.status = 'closed'`, `pool.tokenStatus = 'burned'`, `pool.distributionStatus = 'completed'`, `pool.burnTxSignature = txSig`
      3. If on-chain burn fails (e.g., no mint authority), log error to Sentry via `errorMonitor.captureException()` and still close the pool in MongoDB with `pool.tokenStatus = 'burn_pending'` -- admin can retry burn manually via a separate admin-only POST with `action: 'retry-burn'`.

    Add admin-only POST with `action: 'sweep-unclaimed'`:
    - Verify admin via `getAdminConfig().isAdmin(adminWallet)`
    - Verify `claimWindowExpiresAt` has passed
    - Sum unclaimed amounts, record as treasury deposit
    - Call `finalizePool()` (which includes on-chain burn per above)

    GET endpoint (Vercel cron calls GET):
    1. Verify `CRON_SECRET` -- check `req.headers.authorization === 'Bearer ' + process.env.CRON_SECRET'`. Return 401 if invalid.
    2. `dbConnect()`
    3. Query all pools where `bagsTokenMint` exists AND `graduated` is false AND `status` is in `['open', 'funded', 'filled']`
    4. For each pool, call Bags API: `GET https://public-api-v2.bags.fm/api/v1/token/{bagsTokenMint}` with header `x-api-key: process.env.BAGS_API_KEY`
    5. Parse response -- check if token status indicates graduation (look for `status === 'MIGRATED'` or `graduated === true` in Bags response)
    6. If Bags says graduated but MongoDB doesn't:
       - Update pool: `graduated = true`, `graduatedAt = new Date()`, `bagsTokenStatus = 'MIGRATED'`
       - If Bags response includes `marketCap`, set `graduationMarketCap`
       - If Bags response includes `priceUSD`, set `graduationPriceUSD`
       - Log to Sentry: `errorMonitor.captureMessage('Reconciliation: Pool graduation detected', { extra: { poolId, bagsTokenMint } })`
       - Trigger Squad DAO creation by calling `/api/pool/finalize` internally (same pattern as webhook handler in bags.ts -- `triggerSquadCreation`)
    7. Return summary: `{ checked: N, reconciled: N, errors: N }`

    Wrap handler with `withErrorMonitoring`.
