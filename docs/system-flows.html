Marketplace Escrow Sale

Fixed USD Price · USDC Settlement · Squads Multisig Gating

PHASE 1 — LISTING
01 Vendor Creates Asset VENDOR
Upload photos, fill specs. AI watch analysis auto-fills brand, model, condition, estimated price.
POST /api/ai/analyze-watch
02 Admin Approves Sale Request ADMIN
Admin reviews asset authenticity and approves the listing for minting.
POST /api/nft/approveSale
03 Mint NFT + Create Escrow SQUADS
Creates Squads proposal for initialize instruction. NFT locked in vault, USDC vault created.
POST /api/escrow/create-with-mint
Escrow PDA created with USDC mint as payment token
NFT transferred to escrow nft_vault
Empty USDC vault initialized (awaiting buyer deposit)
sale_price set in USDC atomic units (6 decimals)
status: 'listed' listingPriceUSD: 15000 listingPrice: 15000000000 (USDC atomic) paymentMint: 'USDC'
PHASE 2 — PURCHASE
04 Buyer Selects Payment Token BUYER
Price shown as $15,000 USD (fixed). Buyer chooses to pay with USDC or SOL. If SOL: Jupiter ExactOut quote calculates how much SOL needed for exact USDC amount.
05 Deposit USDC to Escrow BUYER
If USDC: Direct transfer from buyer's ATA to escrow vault.
If SOL: Jupiter swap (SOL → USDC) into buyer's ATA, then transfer to vault.
exchange instruction → POST /api/escrow/purchase
status: 'funded' buyerWallet: '8N3b...' buyerShippingAddress: { fullName, street, city, ... } fundedAmount: 15000000000 paymentMint: 'USDC' | 'SOL' swapTxSignature: '...' (if SOL)
PHASE 3 — SHIPMENT
06 Vendor Ships Item VENDOR
Submit tracking carrier, number, and proof photos (uploaded to IPFS).
POST /api/escrow/submit-shipment
07 Admin Verifies Shipment ADMIN
Reviews tracking + proof photos. Approves or rejects (vendor must resubmit).
POST /api/escrow/verify-shipment
PHASE 4 — DELIVERY & SETTLEMENT
08 Buyer Confirms Delivery BUYER
Buyer confirms receipt, optional 1-5 rating and review.
POST /api/escrow/confirm-delivery
09 Squads Releases Funds SQUADS
confirm_delivery instruction executed via Squads multisig.

ALT Admin Initiates Refund ADMIN
If shipment rejected or dispute: refund_buyer instruction returns full USDC to buyer, NFT to seller.

100% USDC → Buyer (full refund)

Fractional Ownership Pools

Crowdfunded RWA · USDC Settlement · Bags DEX Trading

PHASE 1 — POOL CREATION
01 Create Pool ADMIN
New pool from dealer asset, or convert existing escrow listing to fractional pool. Sets total shares, price per share, target amount, and liquidity model.
POST /api/pool/create
POST /api/pool/convert-from-escrow
status: 'open' totalShares: 1,000,000 sharePriceUSD: $0.10 targetAmountUSD: $100,000 liquidityModel: 'p2p' | 'amm' | 'hybrid'
PHASE 2 — INVESTOR FUNDING
02 Investors Buy Shares INVESTOR
Direct investment or via Bags bonding curve. Ownership % auto-calculated per investor. Pool status changes to filled when all shares sold.
POST /api/pool/invest
POST /api/pool/buy (Bags bonding curve)
participants: [{ wallet, shares, investedUSD, ownershipPercent }] sharesSold: 800,000 / 1,000,000 fundsInEscrow: $80,000
PHASE 3 — VENDOR PAYMENT
03 Pay Vendor via Squads SQUADS
When pool fills, admin triggers vendor payment. Requires custody verification first (escrow protection).
POST /api/pool/pay-vendor
97% → Vendor ($97,000 USDC)
3%
PHASE 4 — CUSTODY TRANSFER
04 Vendor Ships to LuxHub Vault VENDOR
Vendor ships physical asset to LuxHub secure storage.
POST /api/pool/custody (action: submit_tracking)
05 Custody Pipeline ADMIN
5-stage verification pipeline. Admin authenticates physical asset and stores in vault.
pending → shipped → received → verified → stored
custodyStatus: 'stored' status: 'active' (asset in LuxHub custody) custodyProofUrls: [IPFS photos] custodyVerifiedBy: adminWallet
PHASE 5 — SECONDARY TRADING
06 Token Holders Trade Shares INVESTOR
P2P direct trades between holders, or automated via Bags bonding curve AMM. Price adjusts with demand.
POST /api/pool/buy (secondary market)
PHASE 6 — RESALE
07 List Asset for Full-Price Resale ADMIN
Asset listed at $120,000. Projected ROI calculated for investors. Full-price buyer purchases the physical asset.
POST /api/pool/list-for-resale

State Machines

Status transitions for Escrow, Pool, Shipment, Custody, and Offers

Escrow Status

initiated → listed → funded → shipped → delivered → released
funded → refund → cancelled
initiated → converted (to pool)

Pool Status

open → filled → funded → custody → active → listed → sold → distributed → closed

Shipment Status

pending → shipped → in_transit → delivered → proof_submitted → verified
proof_submitted → rejected (resubmit)

Custody Status (Pools)

pending → shipped → received → verified → stored

Offer Status

pending → accepted → settled
pending → rejected / withdrawn / expired
accepted ↔ countered

Data Tracked Per Transaction

Every piece of real-world data managed by the platform

Escrow Data

Listing Data
Asset (brand, model, serial, condition, specs, IPFS photos) · Pricing (listingPriceUSD fixed, USDC atomic) · Sale mode (fixed, offers, crowdfunded) · On-chain (escrowPda, nftMint, seed, bump) · Squads (transactionIndex, proposedAt)
Purchase Data
Buyer wallet · Payment token (SOL or USDC) · Jupiter swap tx (if SOL) · Funded amount · Shipping address (name, street, city, state, zip, country, phone, email, delivery instructions)
Shipment Data
Carrier (FedEx/UPS/DHL/USPS) · Tracking number · Auto-generated tracking URL · IPFS proof photos · Ship-from address · ETA · Vendor notes · Rejection history (reason, admin, previous tracking)
Delivery & Settlement
Confirmation (who, type, when) · Rating (1-5) · Review text · Actual delivery date · Squads proposal index · Financial split (97% vendor, 3% treasury) enforced on-chain
Refund Data
Cancel reason · Refunded amount (full USDC) · Cancel/propose/execute timestamps · Squads proposal index · Both parties notified automatically

Pool Data

Pool Configuration
Asset reference · Total shares · Share price · Target amount · Liquidity model (P2P/AMM/Hybrid) · Bonding curve type · Min buy-in · Max investors · Source type (dealer/escrow conversion)
Investor Records
Participants array (wallet, shares, investedUSD, ownershipPercent, date) · Shares sold · Funds in escrow · Investor count · Auto-calculated ownership %
Tokenization (Bags)
SPL token mint address · Token status (pending/minted/unlocked) · Bonding curve active · Current price · Curve type (linear/exp/sqrt)
Custody Chain
5-stage pipeline (pending → shipped → received → verified → stored) · Tracking carrier + number · IPFS proof photos · Admin who verified · All timestamps recorded
Vendor Payment
Payment amount (97% of target) · Payment timestamp · On-chain tx signature · Squads proposal reference
Resale & Distribution
Resale listing price · Sold price · Buyer wallet · Per-investor distributions (amount, profit, ROI) · Squads distribution proposal · Royalty amount (3%)
